# PolyLang 価格設定分析 v2

## 設計思想

PolyLangは「文法に頼らない言語学習」がコンセプト。価格設計もこの哲学に連動させる。

- **コア機能**（安い・気軽に）: Audio, Explorer, Correction → アウトプット練習を促進
- **便利ツール**（中程度）: Extract → 特定場面でのみ使う補助機能
- **補助輪**（高い・抵抗あり）: Explanation → 文法解説に頼る行為に心理的コストを設ける

課金モデルは **単品購入 + サブスクリプション** のデュアル構造。
サブスクの方が圧倒的にお得に見せることで、月額課金への転換を促す。

## AIモデル選択

機能ごとに最適なモデルを使い分け、コストと品質を両立。

| モデル | Input | Output | 用途 |
|--------|-------|--------|------|
| GPT-5.2（左） | $1.75/1M | $14.00/1M | 品質重視の機能 |
| GPT-4o-mini（中） | $0.15/1M | $0.60/1M | コスト重視の機能 |

| 機能 | 採用モデル | 理由 |
|------|----------|------|
| Audio | TTS（固定） | 音声合成は別API |
| Explorer | **mini** | 単語探索は高品質不要 |
| Correction | **GPT-5.2** | 添削は品質最重要 |
| Explanation | **GPT-5.2** | 文法解説は品質重要 |
| Extract | **mini** | 画像OCRはminiで十分 |
| Tokenize | **mini** | 分割処理のみ |

## 実装要件

### Stripe連携
- 単品購入: Stripe Checkout（one-time payment）
- サブスク: Stripe Subscriptions（monthly recurring）
- Webhook: 支払い完了時にクレジット付与 / プラン有効化

### データベース
- `profiles` テーブルに `plan` カラム追加（free / standard / pro）
- `plan_expires_at` で有効期限管理
- 日次上限チェック: `daily_usage` テーブル活用（既存）
- 月次上限チェック: Extract / Explanation の月間使用回数を追跡

### クレジット消費ロジック
- 無料 / サブスクユーザー: 日次・月次上限チェック → 上限内なら無料消費
- 単品購入: Stripe決済完了 → 該当機能を1回実行
- 既存の `checkAndConsumeCredit()` を拡張して上限チェックに対応
